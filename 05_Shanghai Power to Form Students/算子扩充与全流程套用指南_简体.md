### 核心数据链(一定要记住)

```
config.SLUG
   └─▶ data/<slug>/buildings.parquet          (真实楼:footprint + AI 实测高度)
          └─▶ C.assign_all(读 shanghai_lookup.yaml)   → 每栋贴一个 stakeholder(state/developer/resident/…)
                 ├─▶ 主册路：C.scenario_heights(读 power_scenarios.yaml)  只调「高度」
                 └─▶ 进阶路：C.to_recs(df) → recs 列表 ──▶ operators.py（★算子在这★）
                                                             └─▶ apply_regime(读 regimes.yaml)
                                                                    └─▶ plots / measure / 06
```

**recs** 就是算子的工作单位——一个 list,每个元素是一栋楼的 dict:

```python
rec = {
    "geom":   <shapely Polygon/MultiPolygon>,   # footprint,单位「米」，EPSG:32651
    "h":      <float>,                          # 高度(米)
    "sh":     "state"|"developer"|"resident"|"unknown"|"informal",  # 权利方
    "area":   <float>,                          # footprint 面积(m²)
    "frozen": <bool>,                           # 被 freeze 锁定就别动
}
```

> **GFA 约定**:某栋 GFA ∝ `geom.area × h`。想「守恒」→ 缩面积时把高度补回来(`h /= ratio`);想「加量」→ 只抬 `h`。

## 1. 算子的契约(规则,违反就会坏)

一个算子 = **纯函数**:吃一批楼 `recs`,吐一批新楼 `recs`。

```python
def my_op(recs, target, ...):
    return new_recs
```

五条铁律:

1. **不要改原始 `recs`**:先 `out = [dict(r) for r in recs]`,只改 `out`,返回 `out`。
   (否则前一个算子的结果被污染,配方 `steps` 串起来会出错。)
2. **只动 target、未 frozen 的楼**:判断用 `r["sh"] in target and not r.get("frozen")`。
3. **几何要合法**:缩太狠会变空多边形。`aff.scale` / `aff.rotate` 比 `buffer` 安全。
4. **单位是米**(EPSG:32651),不是经纬度。
5. **想清楚 GFA**:守恒(缩面积补高度)还是加量(只抬高度)。

## 2. 三步基本套用(只在 05 内)

### 步骤 1 · 写

复制 `engine/my_operator.py` 里的 `taper` 当模板,改中间几行;或直接在 notebook 贴:

```python
import shapely.affinity as aff

def my_op(recs, target, amount=1.5):
    """一句话:这个算子对目标楼做了什么形态操作。"""
    out = [dict(r) for r in recs]                       # ① 复制
    for r in out:
        if r["sh"] in target and not r.get("frozen"):  # ② 只动目标、未冻结
            r["h"] = r["h"] * amount                    # ③ 你的形态操作
    return out
```

### 步骤 2 · 登记

```python
import operators as ops
ops.register("my_op", my_op)      # 现在 "my_op" 与内置 9 算子完全平权
```

> ⚠️ **`ops.register()` 只活在当前 notebook kernel**——它是 runtime 登记,重开就没了,**而且传不到 `06`**(见 §3 的关键警告)。要永久、要到处都能用,请走 §4 的 checklist。

### 步骤 3 · 写进配方 `regimes.yaml`

```yaml
my_regime:
  label: 我的权力体制
  desc: 先锁政府,再用我的算子动居民/开发商,最后加密。
  steps:
    - { op: freeze, who: [state] }
    - { op: my_op, target: [resident, developer], amount: 1.8 }
    - { op: densify, target: [resident, developer], far_gain: 1.4 }
```

`steps` 里每个 `op` = 你登记的名字;其余键 = 传给函数的参数(对上函数签名即可)。

做完这三步,在 notebook **阶段 C** 就能看到你的算子:

- `plots.operator_demo(before, after, "标题")` — 单一算子 before/after
- `plots.regime_compare(recs, after, labels=...)` — 你的体制与其他体制同色阶横比
- `M.compare(...)` + `plots.fingerprint_bars(...)` — 量化形态指纹

## 3. 「套用到全部套用」— 完整传播链

一个算子登记 + 写进某个 regime 后,会流经**两个 app、七个消费点**:

| #   | 消费点                                  | 在哪                                          | 靠什么读到算子                        |
| --- | --------------------------------------- | --------------------------------------------- | ------------------------------------- |
| 1   | `operator_demo` 单算子 before/after     | 05 notebook 阶段 C                            | 直接调用 `ops.my_op(...)`             |
| 2   | `regime_compare` 四体制横比             | 05 notebook 阶段 C                            | `ops.apply_regime(recs, regs[name])`  |
| 3   | `measure` + `fingerprint_bars` 形态指纹 | 05 notebook 阶段 C                            | 同上(对 after recs 量测)              |
| 4   | `RUN_ALL_SITES` 跑遍全街道              | 05 notebook 阶段 E                            | 每站 `apply_regime` 一次              |
| 5   | `massing_*.jpg` 体块参考图              | 06 `engine/massing.py`                        | `ws05.regime_recs` → `apply_regime`   |
| 6   | `canvas.html` 互动 3D                   | 06 `engine/build_canvas.py`                   | 同上                                  |
| 7   | `report.html` AI 渲染报告               | 06 `engine/build_report.py` + `prompt_gen.py` | 同上 + 读 `06/config.yaml` 的 prompts |

`06` 怎么接到 `05` 的算子(`06/engine/ws05.py`):

```python
import operators as ops                            # 直接 import 05 的 operators.py
regs = ops.load_regimes(WS05 / "regimes.yaml")     # 读 05 的 regimes.yaml
out[name] = ops.apply_regime(recs, regs[name])     # 套每个体制
```

### ⚠️ 关键警告:runtime 登记到不了 06

05 notebook 阶段 F 是用 **subprocess** 调用 06:

```python
subprocess.run([sys.executable, "run.py", s], cwd=str(R06))
```

这是**另一个 Python 进程**,会重新 `import operators`——你在 notebook 里的 `ops.register("my_op", ...)` **完全不存在于那个进程**。

因此:

- 只做 §2 的 `ops.register` → 你的算子只在**当前 notebook** 有效(消费点 1–4 可以,消费点 5–7 会 `KeyError: 'my_op'`)。
- 要真正「套用到全部套用」→ **必须把算子永久写进 `engine/operators.py` 的 `OPS` dict**,让任何进程 import 都拿得到。

## 5. 完整范例:写 `twist`(权力漩涡)算子,从 0 跑到 06

一个**全新、GFA 守恒、几何稳健**的算子:目标楼 footprint 依「到权力重心的距离」旋转——越外围转越多,像被权力中心卷成漩涡。旋转不改面积、高度不变 → 单栋 GFA 守恒。

### ① 贴进 `engine/operators.py`

```python
def twist(recs, target, deg_per_100m=12.0):
    """权力漩涡:目标楼 footprint 依「到全域 GFA 重心的距离」旋转(越远转越多)。
    旋转不改面积、高度不变 → 单栋 GFA 守恒。deg_per_100m: 每离中心 100m 多转几度。"""
    import math
    import shapely.affinity as aff
    out = [dict(r) for r in recs]                       # ① 复制,别动原始
    A = sum(r["geom"].area * r["h"] for r in out) or 1.0            # 全域 GFA
    Px = sum(r["geom"].centroid.x * r["geom"].area * r["h"] for r in out) / A
    Py = sum(r["geom"].centroid.y * r["geom"].area * r["h"] for r in out) / A   # 权力重心
    for r in out:
        if r["sh"] in target and not r.get("frozen"):  # ② 只动目标、未冻结
            d = math.hypot(r["geom"].centroid.x - Px, r["geom"].centroid.y - Py)
            r["geom"] = aff.rotate(r["geom"], deg_per_100m * d / 100.0, origin="centroid")  # ③ 依距离旋转
    return out
```

文件尾 `OPS` 加一行:

```python
OPS = {..., "scale_graded": scale_graded, "twist": twist}
```

### ② `regimes.yaml` 加体制

```yaml
power_vortex:
  label: 权力漩涡
  desc: 先锁政府,再让居民/开发商楼依到权力重心的距离旋转(越外围转越多),最后小幅加密。
  fingerprint: 旋转错落(平面纹理扭转、天际线微乱)
  steps:
    - { op: freeze, who: [state] }
    - { op: twist, target: [resident, developer], deg_per_100m: 15 }
    - { op: densify, target: [resident, developer], far_gain: 1.3, cap_m: 300 }
```

### ③ `06_AI Render Students/config.yaml`

```yaml

  - current
  - developer_led
  - state_led
  - resident_self_build
  - shared
  - power_vortex # ← 新增
```

## 6. 自检清单 + 常见坑

写算子本身:

- [ ] 返回**新 list**,没有原地改 `recs`?
- [ ] 只动了 `target` 内、`not frozen` 的楼?
- [ ] 改几何后 `geom` 仍是合法多边形?(缩太狠会变空)
- [ ] GFA:守恒(缩面积补高度)还是加量(只抬高度)——想清楚?
- [ ] 单位是米(EPSG:32651)?

套用到全部:

- [ ] 有没有把函数**永久写进 `operators.py` 的 `OPS`**?(只 `ops.register` → 06 会 `KeyError`)
- [ ] `regimes.yaml` 里新体制的 `op` 名字 = `OPS` 的 key?
- [ ] 06 要出这体制 → `06/config.yaml` 的 `regimes:` 清单有加名字?
- [ ] `06/config.yaml` 的 `prompts.regimes.<name>.look` 有写?(没写会 fallback 到 current 的外观)
- [ ] 改完 `operators.py`/`regimes.yaml` 后,05 notebook 顶格重跑(清模块缓存)、06 重跑 `run.py`?

常见坑速查:
| 症状 | 原因 | 解法 |
|---|---|---|
| 06 `KeyError: 'my_op'` | 只做了 runtime `ops.register` | 永久写进 `operators.py` 的 `OPS` |
| 06 没渲染新体制 | `06/config.yaml` 的 `regimes:` 没加名字 | 加进清单 |
| 06 报告外观不对 | 缺 `prompts.regimes.<name>` | 补一个 `look` block |
| notebook 改了没生效 | 模块被缓存 | 重跑顶格(它会 `del sys.modules[...]`) |
| 几何变空 / 出图破面 | 缩放比例太小、`buffer` 负太多 | 用 `aff.scale`/`aff.rotate`、加下限保护 |
